杀戮尖塔2 地图区域影响哪些随机结果 ​

这篇回答一个很具体的问题：在 STS2 里，密林、暗港、巢穴、荣耀这些地图区域，会不会改变问号、事件、怪物和 Boss 的随机结果？

简短版：

  • 地图区域会改变：地图长度、问号数量、事件池、普通怪池、精英池、Boss 池、先民池、背景和音乐。
  • 地图区域不直接改变：问号翻成“事件 / 普通战斗 / 商店 / 宝箱”的基础 odds。
  • 所以你会感觉“不同地图的问号不一样”，主要原因不是问号概率表换了，而是问号数量不同，且问号变成事件后抽的是当前区域的事件池。

先分清两个“地图” ​

玩家讨论“地图”时，通常混着说两件事：

说法实际指什么会影响什么
“这一幕是密林 / 暗港 / 巢穴 / 荣耀”区域，源码里是 act房间数、问号数、事件池、怪物池、Boss 池等。
“这个问号会变成什么”问号节点翻开时的房间类型 roll使用一套全局基础 odds，并随本局问号历史动态变化。

后面所有结论都按这两层拆开讲。

区域影响事件池 ​

问号房先 roll 房间类型。只有当它 roll 成“事件”后，游戏才会从事件队列里取具体事件。

这个事件队列不是全游戏事件混在一起抽，而是：

text
当前区域专属事件 + 共享事件 - 尚未解锁的事件

这里的“共享事件”还要再分两种：

  • 区域列表复用：某个事件被写进多个区域的事件列表。当前明显的例子是 沉没雕像，它同时在密林和暗港。
  • 全区域共享事件：ModelDb.AllSharedEvents 里的事件，会追加到每个区域的事件池里。比如水晶球、茶艺大师、遗物交换商等。

也就是说，在密林走问号时，如果问号变成事件，候选池大致是“密林区域事件 + 全区域共享事件”；在暗港则是“暗港区域事件 + 全区域共享事件”。如果事件还没解锁，或当前状态不满足事件出现条件，游戏会跳过它。

下面的事件图来自本地游戏资源导出的事件插图；灯火钥匙没有标准事件背景图，这里用的是游戏里的灯火钥匙任务图。

密林区域事件 ​

密林的事件池偏“植物、异鸟、森林遗迹、泉水、迷路与变化”。其中 沉没雕像 也会在暗港出现。

暗港区域事件 ​

暗港的事件池偏“水、宝藏、商店式交易、构装体、漩涡与沉没遗迹”。其中 沉没雕像 与密林共享。

回复 / 最大生命换伤害
深渊浴场

敬而远之回复；投身池水获得最大生命但受伤，可继续沉溺。

药水 / 遗物换最大生命
淹水灯塔

装瓶拿药水；攀爬拿遗物但失去最大生命。

金币消费 / 多种小奖励
无尽传送带

观察主厨可升级牌；花金币吃不同菜，可能获得金币、药水、变化、升级、最大生命等。

遗物 / 战斗
重拳出击

偷拿遗物会得到伤口；也可以直接开战换更好奖励。

回复 / 附魔 / 遗物加诅咒
螺旋漩涡

饮用回复；观察给打击或防御附魔；伸手拿遗物但加入负面牌。

金币 / 遗物
沉没雕像

潜水拿金币但掉血，或拿石剑遗物。也在密林事件池中。

金币 / 诅咒换更多金币
淹水金库

开小箱拿少量金币；开大箱拿更多金币但得到贪婪。

血液墨水给最大生命；付金币给一张或多张牌加稳定附魔。

巢穴区域事件 ​

巢穴的事件池偏“虫群、寄生、融合、灵魂、异变与跨角色牌”。整体更常见“拿强奖励，但同时改牌组结构或付出代价”。

删基础牌 / 获得融合牌
熔合者

移除 2 张打击或 2 张防御，换一张对应强牌。

获得卡牌
害虫杀手

在两张虫害主题卡牌之间选一张加入牌组。

跨角色牌 / 棱镜碎片
色彩哲学家

选择角色颜色拿对应角色卡牌，或选择“平等”拿棱镜碎片。

金币 / 高风险遗物
巨大花卉

逐步深入会持续掉血；可中途拿金币，也可摸到核心遗物。

附魔 / 删牌加诅咒
人形洞穴之地

随机能力 / 随机 0 费牌
被寄生的自动机械

研究拿随机能力牌；触碰核心拿随机 0 费牌。

金币 / 诅咒换遗物

金币 / 战斗
灯火钥匙

交还钥匙拿金币；留下钥匙会进入战斗。它还会影响共享事件“战史学家 付袭”。

花金币换删牌或加牌
修禅织网者

付金币学习技巧，常见结果包括删 1-2 张牌，或加入开悟等。

荣耀区域事件 ​

荣耀的区域事件最少，内容更偏“后期高波动”：战斗检测、复制牌组、判决、定制卡牌、强遗物和诅咒交换。

限时战斗 / 奖励档位
战痕累累的训练假人

遗物 / 诅咒加附魔

镜中倒影 影倒中镜

全回复遗物 / 掉血拿遗物
圆桌茶会

判决事件 / 多分支奖励

全区域共享事件 ​

这些事件来自全局共享事件列表，会追加到每个区域的事件池里。实际能不能出现，还会受解锁进度、当前牌组、金币、药水、遗物、生命值等条件影响。

卡牌奖励 / 掉血换无色牌

付费占卜 / 诅咒占卜

最大生命 / 攻击牌附魔

区域影响怪物、精英和 Boss ​

每个区域都有自己的遭遇池：

  • 普通战斗取当前区域的普通怪队列。
  • 精英战斗取当前区域的精英队列。
  • Boss 从当前区域 Boss 池里选；如果还有没见过的 Boss，会优先按发现顺序安排。

这就是为什么同样是普通战斗图标，密林和暗港的敌人完全不同；同样是精英图标，背后的候选战斗也按区域换。

区域影响地图长度和问号数量 ​

区域还会改变这一幕有多少房间、多少问号、多少休息点：

区域基础房间数问号节点数量休息点数量
密林15约 10-14约 6-7
暗港15约 10-14约 6-7
巢穴14约 9-13约 6-7
荣耀13约 9-13约 5-6

多人模式下，每幕基础房间数会再少 1。

这点很重要：问号“翻成什么”的基础规则即使相同，区域也能通过问号数量改变你实际遇到事件、怪物、商店、宝箱的次数。

区域影响地图节点分布 ​

标准地图大致有这些固定规则：

  • 第一排普通节点固定是普通战斗。
  • 接近 Boss 前的一排固定是休息点。
  • 中段有一排固定是宝箱；某些效果可能把这排宝箱替换成精英。
  • 商店默认 3 个。
  • 精英默认 5 个；高进阶的“蜂拥精英”会增加精英数量。
  • 其余节点按区域配置补上休息点、问号、商店、精英，剩余空位变成普通战斗。

游戏还会避免一些不舒服的排布，例如精英、休息点、宝箱、商店等不会轻易连续贴在父子节点上，休息点和精英也不会出现在太靠下的位置。

问号概率不是按区域换表 ​

问号房翻开时，基础 odds 大致如下：

结果基础 odds
普通战斗10%
宝箱2%
商店3%
事件剩余概率，初始约 85%
精英默认关闭

这里容易误解：它不是每次都固定“85% 事件”。如果某个非事件结果这次没 roll 到，它的 odds 会增加；如果 roll 到了，它会回到基础值。事件概率是剩余概率，所以会随着怪物、宝箱、商店 odds 的累计而下降。

直观理解：连续几个问号都出事件后，后续问号出普通战斗、商店、宝箱的机会会逐步变高。出过一次普通战斗后，普通战斗 odds 又会回到基础值。

这套问号 odds 目前不是“密林一张表、暗港一张表、巢穴一张表”。区域影响的是两个更外层的东西：这幕地图有多少问号；问号变成事件后，从哪个事件队列取具体事件。

路线规划时怎么用 ​

  • 想多找区域专属事件，就多走当前区域问号；但问号不保证都是事件。
  • 巢穴和荣耀的问号节点数量通常比密林、暗港少一点，所以“刷问号”的空间也少一点。
  • 如果前面问号连续出事件，后面问号变成怪物、商店或宝箱的概率会变高。
  • 不同区域的精英、Boss 和普通怪池不同，不能只看节点图标判断难度。
  • 解锁进度会改变事件池和先民池；新档、老档、多人、Mod 环境都可能让结果不同。

底层实现原理 ​

这一节是实现细节，不感兴趣可以跳过。前面已经覆盖路线规划需要知道的内容。

源码层面，区域由 ActModel 派生类定义。当前常见区域包括：

区域类主要配置
密林Overgrowth15 个基础房间、13 个专属事件、3 个弱战斗、3 个 Boss 候选。
暗港Underdocks15 个基础房间、10 个专属事件、3 个弱战斗、3 个 Boss 候选。
巢穴Hive14 个基础房间、10 个专属事件、2 个弱战斗、3 个 Boss 候选。
荣耀Glory13 个基础房间、7 个专属事件、2 个弱战斗、3 个 Boss 候选。

开局默认区域列表是“密林、巢穴、荣耀”。暗港解锁后，有机会替换第一幕；如果暗港已解锁但还没被发现，非多人模式下会优先出现。

生成房间池时，游戏会：

  1. 把当前区域的 AllEvents 和 ModelDb.AllSharedEvents 合并。
  2. 按解锁进度移除还没揭示的事件。
  3. 洗牌后放入本幕事件队列。
  4. 分别生成普通战斗队列、精英战斗队列、Boss、先民。
  5. 进入事件房时，取事件队列里的下一个有效事件，并记录已访问事件，尽量避免重复。

标准地图由 StandardActMap 生成。地图 RNG 使用 act_N_map 从本局种子派生，因此同一种子、同一幕、同一游戏状态下，地图结构和节点分布更容易复现。

问号房 odds 由 UnknownMapPointOdds 维护。基础值是普通战斗 0.10、宝箱 0.02、商店 0.03、精英 -1，事件概率用 1 - 正数非事件 odds 之和 计算。每次问号 roll 后，roll 到的类型重置为基础值，没 roll 到的可用类型增加自己的基础值。
